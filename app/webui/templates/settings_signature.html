Signatur

Header der gesetzten Mails kennzeichnet (verhindert Doppel-Injektion)
Benötigt Mail.ReadWrite.All App-Permission in der Entra App-Registrierung
Wenn die Gateway-Signatur bereits im Nachrichtenbereich erkannt wird (z. B. durch das Outlook Add-in), wird sie nicht erneut injiziert
Wenn eine Gateway-Signatur in einer zitierten Vorgänger-Mail erkannt wird, wird keine neue Signatur eingefügt — verhindert Stacking bei Ping-Pong-Threads
CID = Bild als MIME-Anhang verlinkt; data: URI = Bild direkt im HTML eingebettet
Outlook-Signaturen werden vor der Gateway-Injektion entfernt — verhindert Doppel-Signaturen.
⚠ Auf eigene Gefahr — Feature in Entwicklung: kann Mails im Einzelfall fehlerhaft zuschneiden. Im Zweifel deaktiviert lassen.
