Signaturzertifikate für S/MIME-aktivierte Benutzer

Import per PKCS12-Bundle (.p12 / .pfx) oder automatisch via CASTLE ACME (Auto-Enroll). Ausgehende Mails werden automatisch signiert, sobald ein Standardzertifikat für den Absender vorliegt. Private Schlüssel werden per Dateisystemberechtigungen (600) geschützt und können zusätzlich verschlüsselt abgelegt werden (Einstellungen → S/MIME). Empfohlen: Schlüssel in Azure Key Vault migrieren — dort verlässt der private Schlüssel nie den HSM-gesicherten Speicher; das Gateway signiert über die KV-API. Benutzer werden unter Einstellungen → Postfach-Konfiguration für S/MIME aktiviert.
