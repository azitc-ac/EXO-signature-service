Backup erstellen

Erstellt ein vollständiges ZIP-Archiv aller kritischen Gateway-Daten — geeignet für Migrationen, Umzüge auf neue Server oder als regelmäßige Sicherungskopie.

Im Backup enthalten
  • settings.json — Konfiguration
  • auth.pfx — EXO-Auth-Zertifikat
  • smime/ — S/MIME Keys & Certs
  • acme/ — ACME Account Keys & Orders
  • mail_audit.db — Audit-Protokoll
  • stats*.json — Statistiken
  • templates/ — Signatur-Vorlagen
Nicht enthalten
  • logs/ — Laufzeit-Logs
  • le-config/ — Let's Encrypt
  • TLS-Zertifikate

Info Migration auf einen neuen Server

Zum Umzug auf eine neue VM (z. B. von Raspberry Pi → Azure):

  1. Backup auf dem alten Server herunterladen (Button oben)
  2. Gateway auf dem neuen Server installieren (docker compose up -d)
  3. Im Web-UI des neuen Servers: Backup hochladen & wiederherstellen
  4. Neu starten (Button „Jetzt neu starten" nach dem Restore, oder docker compose restart)
  5. DNS-Eintrag auf die neue IP umstellen
  6. TLS-Zertifikat neu ausstellen (ACME läuft automatisch nach Neustart)
S/MIME & ACME: Account Keys, Zertifikate und ACME-State werden vollständig migriert — keine manuelle Neuregistrierung nötig.
